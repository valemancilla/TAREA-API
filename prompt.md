Crea una aplicación To-Do List completa con HTML, CSS y JavaScript. La aplicación debe tener una página única con diseño centrado vertical y horizontalmente, usando un fondo con gradiente oscuro que va de gris  a negro oscuro. El contenedor principal debe estar centrado con ancho máximo moderado y seguir un diseño minimalista y moderno con mucho espacio.
En la parte superior coloca el título "Just do it." con fuente sans-serif bold y muy grande en color blanco, centrado y con espaciado generoso debajo. Debajo del título crea un input de tareas con fondo negro ." en color negro, y un botón "I Got This!" integrado a la derecha del input con fondo blanco y texto negro. Ambos elementos deben formar una unidad con bordes muy redondeados, donde el input ocupa la mayoría del ancho y el botón está pegado a la derecha.
Para la lista de tareas, cada tarea debe mostrarse en un contenedor individual con fondo negro o gris muy oscuro, bordes muy redondeados, padding generoso interno y espaciado vertical entre tareas. El texto de cada tarea debe estar a la izquierda en color blanco o gris claro con la etiqueta "opcional" visible, y a la derecha debe haber dos botones circulares blancos: uno con ícono de check para marcar como completada y otro con ícono de papelera para eliminar.
La funcionalidad JavaScript debe estar conectada a MockAPI mediante el método GET con fetch para cargar las tareas al iniciar la aplicación. Debe permitir agregar nuevas tareas al hacer clic en "I Got This!" o presionar Enter, limpiando el input después de agregar. El botón de check debe marcar o desmarcar tareas opcionalmente tachando el texto o reduciendo su opacidad, y el botón de eliminar debe remover la tarea de la lista con animación suave. La aplicación debe realizar las peticiones HTTP correspondientes (POST para crear, PUT o PATCH para actualizar, DELETE para eliminar) a la API de MockAPI.
Agrega transiciones suaves al agregar o eliminar tareas, efectos hover en todos los botones con cambio de opacidad o brillo, y animación fade-in al agregar nuevas tareas. La aplicación debe ser responsiva: en móvil reduce ligeramente el tamaño del título manteniendo la estructura, y en tablet y desktop mantén el contenedor centrado con ancho máximo. Los inputs y botones deben adaptarse correctamente en pantallas pequeñas.
El estilo debe ser limpio sin distracciones, con alto contraste entre elementos claros y fondo oscuro, bordes redondeados en todos los elementos interactivos, espaciado consistente y generoso, y fuentes sans-serif modernas. Genera el código completo en un solo archivo HTML con CSS y JavaScript integrados, listo para usar. Incluye un placeholder para la URL de MockAPI que pueda ser fácilmente reemplazado por el usuario.
