Desarrolla una aplicación completa de lista de tareas utilizando HTML, CSS y JavaScript. La interfaz debe consistir en una vista única con elementos alineados al centro tanto vertical como horizontalmente, empleando un fondo degradado oscuro que transiciona desde tonos grises hasta negro profundo.
El contenedor central debe mantener un ancho máximo equilibrado, siguiendo principios de diseño minimalista contemporáneo con amplios márgenes. En la sección superior, incluye el encabezado "Just do it." utilizando tipografía sans-serif en peso bold, tamaño considerable, color blanco, centrado y con espaciado abundante inferior.
Bajo el encabezado, implementa un campo de entrada para tareas con fondo negro, y un botón "I Got This!" posicionado adyacente al lado derecho del input, con fondo blanco y tipografía negra. Ambos componentes deben conformar una unidad visual cohesiva con esquinas muy curvadas, donde el campo ocupa la mayor parte del espacio horizontal y el botón permanece adherido al extremo derecho.
Para el listado de tareas, cada elemento debe renderizarse en un contenedor independiente con fondo negro o gris muy oscuro, esquinas pronunciadamente redondeadas, relleno interno generoso y separación vertical entre ítems. El contenido textual de cada tarea debe alinearse a la izquierda en color blanco o gris claro con la etiqueta "opcional" visible, mientras que en el extremo derecho deben aparecer dos botones de forma circular en color blanco: uno con ícono de verificación para marcar como finalizada y otro con ícono de contenedor de basura para remover.
La lógica JavaScript debe conectarse con MockAPI mediante solicitudes GET usando fetch para recuperar las tareas durante la inicialización de la aplicación. Debe facilitar la adición de nuevas tareas al activar "I Got This!" o mediante la tecla Enter, vaciando el campo de entrada posteriormente. El botón de verificación debe alternar el estado de las tareas, opcionalmente aplicando tachado al texto o reduciendo su opacidad, y el botón de eliminación debe suprimir la tarea de la lista con transición fluida.
La aplicación debe ejecutar las peticiones HTTP apropiadas (POST para creación, PUT o PATCH para actualización, DELETE para supresión) hacia la API de MockAPI. Incorpora transiciones suaves durante la adición o remoción de tareas, efectos hover en todos los botones modificando opacidad o luminosidad, y animación fade-in al agregar nuevos elementos.
La solución debe ser adaptable: en dispositivos móviles, reduce moderadamente el tamaño del encabezado preservando la estructura; en tablets y escritorio, mantén el contenedor centrado con ancho máximo. Los campos de entrada y botones deben ajustarse adecuadamente en pantallas reducidas.
El estilo debe ser depurado sin elementos distractores, con alto contraste entre componentes claros y fondo oscuro, bordes curvados en todos los elementos interactivos, espaciado uniforme y amplio, y fuentes sans-serif contemporáneas.
Genera tres archivos separados: index.html para la estructura, style.css para los estilos, y app.js para la funcionalidad, todos listos para implementación inmediata. Incluye un marcador de posición para la URL de MockAPI que pueda reemplazarse fácilmente por el usuario.
